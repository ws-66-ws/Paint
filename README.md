# Paint

## 程序整体架构概述(vs2026、Windows SDK 10)

Paint 是一个基于 Win32 API 的绘图应用程序，采用消息驱动架构。用户的操作（鼠标、键盘、菜单）会触发消息，由窗口过程 WndProc 根据不同消息类型执行相应逻辑。

## 绘图核心机制

### 数据模型
所有图形统一存储为 Shape 结构，包含类型（自由画笔/线段/矩形）、颜色、线宽、起点、终点或点集。

### 双缓冲绘制
先在内存 DC 中绘制所有图形，再一次性复制到屏幕，消除闪烁。

### 橡皮筋预览
绘制线段/矩形时，鼠标移动过程中用虚线显示临时图形，释放鼠标后转为实线并永久保存。

### 局部重绘
只重绘发生变化的小区域，提高性能。


## PostMessage 与 SendMessage 区别
函数|特性
|----|----|
PostMessage	|异步：把消息放入队列后立即返回，不等待处理结果。适合通知类消息。
SendMessage	|同步：直接调用窗口过程，等待处理完成并返回结果。适合需要返回值或立即生效的操作。
